## Issue Resolution Workflows

### Crawl Error Resolution

#### 404 Errors

1. **Identify source** (internal link, external link, sitemap)
2. **Determine action:**
   - If page existed: Create 301 redirect to relevant page
   - If typo/error: Fix the linking page
   - If external link: Request update or create redirect
3. **Implement fix**
4. **Verify in GSC** (re-crawl URL)
5. **Document resolution**

#### 500 Errors

1. **Check server logs**
2. **Identify cause** (plugin, code error, server issue)
3. **Fix root cause**
4. **Test page loads correctly**
5. **Request re-crawl in GSC**
6. **Monitor for recurrence**

#### Redirect Issues

1. **Identify redirect chains** (3+ redirects)
2. **Map current redirect path**
3. **Update to direct redirect**
4. **Test redirect works**
5. **Update internal links to final URL**

### Index Coverage Resolution

#### "Submitted URL not indexed"

1. **Check page quality** (thin content, duplicate?)
2. **Verify no noindex tag**
3. **Check robots.txt**
4. **Improve content if needed**
5. **Request indexing in GSC**
6. **Monitor status**

#### "Duplicate without canonical"

1. **Identify canonical version**
2. **Add canonical tag to duplicates**
3. **Update internal links to canonical**
4. **Submit canonical to GSC**

### Page Speed Resolution

#### Large Contentful Paint (LCP) Issues

| Issue | Solution |
|-------|----------|
| Large images | Compress, use WebP, lazy load |
| Slow server response | Upgrade hosting, use CDN |
| Render-blocking resources | Defer/async JavaScript, inline critical CSS |
| Client-side rendering | Pre-render, SSR |

#### Cumulative Layout Shift (CLS) Issues

| Issue | Solution |
|-------|----------|
| Images without dimensions | Add width/height attributes |
| Ads/embeds without space | Reserve space with CSS |
| Dynamic content | Reserve space, use transforms |
| Fonts causing shifts | Preload fonts, use font-display |

#### Interaction to Next Paint (INP) Issues

| Issue | Solution |
|-------|----------|
| Long tasks | Break up JavaScript |
| Large DOM | Simplify page structure |
| Heavy JavaScript | Code split, lazy load |
| Main thread blocking | Use web workers |

## Monitoring Alerts Setup

### Critical Alerts (Immediate Action)

| Alert | Threshold | Action |
|-------|-----------|--------|
| Site down | Any | Immediate investigation |
| Traffic drop | >50% day-over-day | Investigate algorithm/technical issues |
| Manual action | Any | Immediate review and remediation |
| Security issue | Any | Immediate fix |

### Warning Alerts (24-Hour Action)

| Alert | Threshold | Action |
|-------|-----------|--------|
| Traffic drop | >20% week-over-week | Investigate |
| New crawl errors | >10 new errors | Review and fix |
| Core Web Vitals fail | Any page failing | Prioritize fix |
| Lost backlinks | >5 important links | Outreach/recovery |

### Monitoring Alerts (Weekly Review)

| Alert | Threshold | Action |
|-------|-----------|--------|
| Ranking drops | >5 positions for top keywords | Content/link review |
| New pages not indexed | After 1 week | Request indexing |
| GBP reviews | New review received | Respond |
| Competitor gains | Top 3 position | Competitive analysis |
