## 5. Content Specifications

### 5.1 Blog Post Structure Template

```markdown
# [Primary Keyword-Rich Title] | Tri-State Aquatic

## Quick Answer / TL;DR
[2-3 sentence summary answering the main question - targets featured snippet]

## Table of Contents
- [Section 1]
- [Section 2]
- [Section 3]
- [FAQ Section]
- [Next Steps]

## [Section 1 - Core Content]
[400-600 words with H3 subheadings]

### [Subsection with related keyword]
[Supporting content]

## [Section 2 - Deeper Dive]
[400-600 words]

## [Section 3 - Local Angle]
[200-400 words connecting to Main Line/Delaware]

## Frequently Asked Questions

### [Question 1 - exact search query format]
[Direct answer, 50-100 words]

### [Question 2]
[Direct answer]

### [Question 3]
[Direct answer]

## Ready to Take the Next Step?
[CTA section with relevant tool/form]

---
**About Tri-State Aquatic Solutions**
[Standard boilerplate]

**Related Articles:**
- [Link to related post 1]
- [Link to related post 2]
- [Link to related post 3]
```

### 5.3 On-Page SEO Requirements

**Every Blog Post Must Include:**

```html
<!-- Title Tag (50-60 chars) -->
<title>[Primary Keyword] | Tri-State Aquatic Solutions</title>

<!-- Meta Description (150-160 chars) -->
<meta name="description" content="[Compelling description with primary keyword and CTA]">

<!-- Open Graph -->
<meta property="og:title" content="[Title]">
<meta property="og:description" content="[Description]">
<meta property="og:image" content="[Featured image URL]">
<meta property="og:type" content="article">

<!-- Schema - Article -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "[Title]",
  "image": "[Featured image URL]",
  "author": {
    "@type": "Person",
    "name": "Brandon Calloway"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Tri-State Aquatic Solutions",
    "logo": {
      "@type": "ImageObject",
      "url": "[Logo URL]"
    }
  },
  "datePublished": "[Publish date]",
  "dateModified": "[Modified date]"
}
</script>

<!-- FAQ Schema (if FAQ section exists) -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "[Question 1]",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[Answer 1]"
      }
    }
  ]
}
</script>
```

**Image Requirements:**
- Featured image: 1200x630px (social sharing optimized)
- In-content images: Compressed, lazy-loaded
- Alt text: Descriptive with keyword where natural
- File names: keyword-descriptive.jpg

### 5.4 Internal Linking Strategy

**Every Post Must Link To:**
- 1 relevant service page
- 1 relevant location page
- 2-3 related blog posts
- 1 interactive tool (calculator, quiz, etc.)

**Link Anchor Text Rules:**
- Use descriptive anchor text (not "click here")
- Vary anchor text naturally
- Include keyword variations
- Link early in content when relevant

- [ ] Provides unique local insight
